# PRD Contract

Before passing PRD to the next agent, verify ALL requirements below.
If any check fails — re-run PM stage, do NOT proceed.

## Output MUST contain:

- [ ] Problem statement (≥2 sentences, explains WHO has the problem and WHY)
- [ ] At least 3 user stories (format: "As a [role], I want [action], so [benefit]")
- [ ] At least 3 functional requirements (numbered, specific, testable)
- [ ] "NOT in scope" section (at least 2 items explicitly excluded)
- [ ] Acceptance criteria with checkboxes
- [ ] Priority label (P0 / P1 / P2)
- [ ] Complexity estimate (S / M / L / XL)

## Output MUST NOT contain:

- [ ] Technical implementation details (no code, no architecture)
- [ ] UI/design specifics (no colors, layouts, components)
- [ ] Database schema or API endpoints
- [ ] Specific library or framework references

## Quality checks:

- [ ] Every user story maps to at least one functional requirement
- [ ] Every acceptance criterion is testable (yes/no answer possible)
- [ ] Scope is realistic for one development cycle
- [ ] No ambiguous words: "fast", "good", "nice", "easy" — replace with measurable criteria
